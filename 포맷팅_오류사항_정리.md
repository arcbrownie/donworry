## 1. 들여쓰기 및 구조 불일치

### ❌ 문제점

- `PersonalRehabilitation2026.tsx` 파일의 최상위 `<div>` 태그가 잘못된 들여쓰기로 되어 있음 (라인 63)
- 반환문의 시작 태그와 끝 태그의 들여쓰기가 일관되지 않음

### ✅ 올바른 형태

```tsx
return (
  <div className="min-h-screen bg-background pb-24 md:pb-0">
    <MainNavigation />
    {/* ... */}
  </div>
);
```

### ❌ 잘못된 형태 (현재)

```tsx
return (
  <div className="min-h-screen bg-background pb-24 md:pb-0">
      <MainNavigation />  // 잘못된 들여쓰기
      {/* ... */}
  </div>
);
```

### 📌 핵심 규칙

- **들여쓰기**: 2칸(또는 프로젝트 설정에 맞게) 일관되게 적용
- **JSX 반환문**: 시작 태그와 닫는 태그의 들여쓰기 일치

## 2. className 조건부 렌더링 스타일

### ❌ 문제점

프로젝트 전반에 걸쳐 조건부 className 사용 방식이 혼재되어 있음

### 발견된 패턴들

#### 1️⃣ 템플릿 리터럴 사용 (일부 파일)
```tsx
className={`font-medium ${result.isEligible ? "text-success" : "text-destructive"}`}
```

#### 2️⃣ cn() 유틸리티 함수 사용 (일부 파일)
```tsx
className={cn("flex items-center", isActive && "bg-primary")}
```

#### 3️⃣ 직접 문자열 연결 (일부 파일)
```tsx
className="flex items-center " + (variant === "debt" ? "text-debt" : "")
```

### ✅ 권장사항

**`cn()` 유틸리티 함수를 일관되게 사용**
- `src/lib/utils.ts`에 이미 정의되어 있으므로 프로젝트 전반에 적용

### 📝 예시

```tsx
// ✅ 좋은 예
import { cn } from "@/lib/utils";

className={cn(
  "base-class",
  condition && "conditional-class",
  variant === "debt" && "debt-variant-class"
)}

// ❌ 피해야 할 예
className={`base-class ${condition ? "conditional-class" : ""}`}
className={"base-class " + (condition ? "conditional-class" : "")}
```

## 3. 빈 줄 일관성

### ❌ 문제점

- 함수/컴포넌트 사이의 빈 줄이 불일치
- 객체/배열 선언 후 빈 줄이 불일치

### ✅ 권장 규칙

| 상황 | 빈 줄 |
|------|-------|
| 컴포넌트 정의 사이 | 1줄 |
| 상수 정의 그룹 사이 | 1줄 |
| useEffect, useState 등 훅 사이 | 없음 |
| 객체/배열 선언 후 | 없음 |

### 📝 예시

```tsx
// ✅ 좋은 예
const blogPosts = [
  { title: "제목1", excerpt: "내용1" },
  { title: "제목2", excerpt: "내용2" },
];

const faq = [
  { question: "질문1", answer: "답변1" },
];

export default function Component() {
  const [state, setState] = useState(0);
  useEffect(() => {
    // ...
  }, []);
  
  return <div>...</div>;
}
```

## 4. 인용부호 일관성

### ❌ 문제점

- 문자열에 작은따옴표(`'`)와 큰따옴표(`"`) 혼용
- HTML 속성에 작은따옴표와 큰따옴표 혼용

### ✅ 권장 규칙

| 상황 | 사용할 인용부호 |
|------|----------------|
| TypeScript/React 코드 | 큰따옴표(`"`) |
| JSX 속성 | 큰따옴표(`"`) |
| 문자열 내부 인용 | 이스케이프하거나 백틱 사용 |

### 📝 예시

```tsx
// ✅ 좋은 예
const title = "2026년 달라지는 개인회생 제도";
document.title = "페이지 제목";
<meta name="description" content="설명" />

// ❌ 피해야 할 예
const title = '2026년 달라지는 개인회생 제도';
document.title = '페이지 제목';
```

## 5. JSX 주석 스타일

### ❌ 문제점

- 주석 스타일이 혼재되어 있음 (한 줄 주석 vs 블록 주석)

### ✅ 권장 규칙

| 상황 | 주석 형식 |
|------|----------|
| JSX 내부 | `{/* 주석 내용 */}` |
| JSX 외부 | `// 주석 내용` 또는 `/* 주석 내용 */` |
| 섹션 구분 | `{/* Section Name */}` |

### 📝 예시

```tsx
// ✅ 좋은 예
export default function Component() {
  // 상태 초기화
  const [state, setState] = useState(0);
  
  return (
    <div>
      {/* Hero Section */}
      <section>
        {/* Main Content */}
      </section>
    </div>
  );
}
```

## 6. 컴포넌트 구조 일관성

### ❌ 문제점

1. `article` 태그 내부 섹션 들여쓰기가 불일치
2. 닫는 태그의 위치가 불일치

### ✅ 권장 구조

```tsx
<article className="flex-1 max-w-4xl prose prose-lg prose-slate max-w-none">
  {/* Section 1 */}
  <section id="section1" className="scroll-mt-20 mb-10">
    <h2 className="text-2xl font-bold text-foreground mb-4">
      제목
    </h2>
    <p className="text-muted-foreground leading-relaxed mb-4">
      내용
    </p>
  </section>

  {/* Section 2 */}
  <section id="section2" className="scroll-mt-20 mb-10">
    {/* ... */}
  </section>
</article>
```

## 7. CSS 클래스 순서 일관성

### ❌ 문제점

- Tailwind CSS 클래스 순서가 파일마다 다름
- 반응형 클래스와 일반 클래스의 순서 불일치

### ✅ 권장 순서 (Tailwind CSS 공식 권장)

1. **Layout** (display, position, float, etc.)
2. **Flexbox/Grid** (flex, grid, etc.)
3. **Spacing** (margin, padding)
4. **Sizing** (width, height, etc.)
5. **Typography** (font, text, etc.)
6. **Visual** (background, border, etc.)
7. **Misc** (opacity, cursor, etc.)

### 📝 예시

```tsx
// ✅ 좋은 예
className="flex items-center justify-between px-4 py-2 text-lg font-bold text-foreground bg-primary rounded-lg hover:bg-primary/90"

// ❌ 피해야 할 예
className="text-foreground px-4 flex items-center bg-primary py-2 rounded-lg"
```

## 8. Props 타입 정의 일관성

### ❌ 문제점

- 인터페이스와 타입 별칭 혼용
- Optional props 표시 방식 불일치

### ✅ 권장 규칙

| 상황 | 사용할 타입 |
|------|------------|
| 컴포넌트 props | `interface` |
| 복잡한 유니온 타입 | `type` |
| Optional props | `?:` 사용 |

### 📝 예시

```tsx
// ✅ 좋은 예
interface BlogCardProps {
  title: string;
  excerpt: string;
  category: string;
  path?: string;
  isPlaceholder?: boolean;
}

// ❌ 피해야 할 예
type BlogCardProps = {
  title: string;
  excerpt: string;
  path?: string;
}
```

## 9. 상수 정의 위치

### ❌ 문제점

- 상수가 컴포넌트 내부와 외부에 혼재

### ✅ 권장 규칙

| 위치 | 사용 시기 |
|------|----------|
| 컴포넌트 외부 | 컴포넌트에서 재사용하지 않는 상수 |
| 컴포넌트 내부 | 컴포넌트 스코프 내에서만 사용하는 상수 |

### 📝 예시

```tsx
// ✅ 좋은 예
const blogPosts = [
  { title: "제목1", excerpt: "내용1" },
];

export default function Component() {
  const [state, setState] = useState(0);
  
  return <div>...</div>;
}
```

## 10. 이모지 사용 일관성

### ❌ 문제점

- 제목에 이모지 사용 여부가 불일치
- 이모지와 텍스트 사이 공백 불일치

### ✅ 권장 규칙

| 상황 | 이모지 사용 |
|------|------------|
| 카드나 위젯 제목 | ✅ 사용 가능 |
| 섹션 제목 | ⚠️ 최소화 (일관성 유지) |
| 이모지와 텍스트 사이 | 공백 없이 사용 |

### 📝 예시

```tsx
// ✅ 좋은 예
<h2>📖 채무조정 완벽 가이드</h2>
<h3>🔧 채무 해결 도구</h3>

// ⚠️ 섹션 제목은 이모지 없이
<h2 className="text-2xl font-bold">채무조정이란?</h2>
```
